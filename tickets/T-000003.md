## Goal
Upgrade `QwenLLMService` to consume OpenRouter's SSE stream and push one `TextFrame` per token chunk so TTS can begin speaking before the full response arrives.

## Acceptance Criteria
- [ ] `QwenLLMService` passes `stream=True` to OpenRouter and reads `data:` SSE lines
- [ ] Each non-empty `delta.content` chunk is pushed as its own `TextFrame` immediately
- [ ] `LLMFullResponseStartFrame` is pushed before the first chunk, `LLMFullResponseEndFrame` after the stream closes
- [ ] Pipeline does not block during streaming (no `await response.text()` accumulation)
- [ ] Interruption mid-stream: when a `CancelFrame` arrives, the in-flight HTTP stream is cancelled cleanly (no dangling tasks)

## Notes
- Use `httpx.AsyncClient.stream()` context manager for SSE
- SSE line format: `data: {"choices":[{"delta":{"content":"..."}}]}`
- End-of-stream sentinel: `data: [DONE]`
- The `CancelFrame` arrives on the upstream direction; the processor must watch for it and cancel the `httpx` request — implement with `asyncio.Task` + `task.cancel()`
- Difficulty: MEDIUM — the async cancellation path is the tricky part

## Implementation Plan
- [ ] Replace single `httpx.post()` with `async with client.stream("POST", ...)` in `QwenLLMService`
- [ ] Parse SSE `data:` lines, skip `[DONE]`, extract `delta.content`
- [ ] Push `TextFrame` per chunk — confirm TTS service receives and starts speaking early
- [ ] Add `CancelFrame` handler: cancel the streaming task and push `LLMFullResponseEndFrame` to clean up
- [ ] Manual test: speak, get partial response, speak again (barge-in) — verify old stream stops
