## Goal
Select the lowest-latency TTS provider supported by Pipecat 1.0.0 and wire it into the pipeline as the audio output stage.

## Acceptance Criteria
- [ ] TTS provider selected with rationale documented (see Notes)
- [ ] Provider's Pipecat service class is imported and instantiated with env-var API key
- [ ] `TextFrame` output from Qwen LLM feeds into TTS service without buffering the full sentence
- [ ] `TTS_API_KEY` (or provider-specific key) added to `.env.example`
- [ ] Audio plays to output transport or is confirmed as `AudioRawFrame` in pipeline output

## Notes
**TTS recommendation: Cartesia** (`pipecat-ai[cartesia]`)
- Lowest latency of any provider in Pipecat's built-ins (~80–120ms TTFB)
- Streams audio chunk-by-chunk as Qwen tokens arrive (true streaming TTS)
- Simple: `CartesiaTTSService(api_key=..., voice_id=...)`
- Alternative: `DeepgramTTSService` — already have the API key, ~150ms TTFB, good quality
- Fallback: `ElevenLabsTTSService` — ~200ms, high quality, separate key required

**Decision**: Use `DeepgramTTSService` first (zero new keys needed), document Cartesia as upgrade path.
